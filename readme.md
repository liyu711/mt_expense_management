设计架构

- 前端：Flask和Jinja
- 后端：Python以及数据处理相关的模块
- 数据库：Sqlite

运行逻辑

1. 根据数据库中已有数据构建前端
2. 用户根据界面操作调用端口进行数据操作
3. 根据变化更新数据库，更新前端
4. 循环

功能介绍

- 基础数据管理
1. 上传和修改PO名称
2. 上传和修改BU名称，并且改变其PO的丛书关系
3. 上传修改项目类型
4. 上传修改用工类型
5. 清空数据库
6. 修改用户信息部分预留了UI但是功能没有实装

- 预测管理
1. 上传，删除和修改项目的名称，类型，从属的PO和BU和IO号。IO号可以一个项目添加多个。项目支持级联删除，会同时删除相关的预测和Capex预测
2. 上传，删除和修改预测的各项数值。上传时需要指定PO，BU，年份和财年。预测的输入分为两部分，人员和非人员预测。非人员预测靠直接输入，人员预测靠输入用工时长同时结合已经输入的用工费率进行计算。修改的
3. 上传，删除和修改Capex预测。上传时需要根据下拉菜单选择已经输入过的PO，BU，财年和项目，然后手动输入描述，预测和cost center。修改的时候可以修改所有的数值。
